Template Websites shows the versatility and flexibility possible with Templates where you're able to develop entire content-rich, database-driven websites, not just without any C# code, but without ever needing to build your App! If your Web App requirements fits within the constraints of a Template Website, there's never been an easier and faster way to develop a Web App in .NET.

Dramatically Simplified

Not having to write any C# code or perform any app builds dramatically reduces the cognitive overhead and conceptual knowledge required for development where the only thing front-end Web developers need to know is Template's syntax and what filters are available to call. All complexity with C#, .NET, namespaces, references, .dlls, strong naming, packages, MVC, Razor, build tools, IDE environments, etc has been eliminated leaving all Web Developers needing to do is run a web .NET Core executable and configure a simple web.settings text file to specify where the website files are and what database they want to connect to (if any). For simple websites without db connectivity the webRoot can be inferred and no web.settings is needed. Deployments are also simplified as no build environment needs to be configured or builds run, you'll just be able to xcopy the web exe's release folder for the target platform and the Template Website directory to the remote server.

Because of Template's high-fidelity with JavaScript, developing a Website with Templates will be instantly familiar to JavaScript devs despite calling and binding directly to .NET APIs behind the scenes.

Ultimate Development Experience

In addition to simplicity, the iterative development experience is unparalleled for a .NET App where for the most part you can just leave the web exe running and add the .html template files needed to build your App. No pre-compilation is required so you can just refresh your page to instantly view modified changes. You'll just need to restart the web exe in order to pick up any config changes.

Extensible Base

Whilst we've initially developed a Self-contained .NET Core App (SCD) to illustrate the concept, the lightweight Template Websites also serves as a good foundation to enhance with any additional .NET Core or ServiceStack features you may need as well as extending its features with custom App filters or Services where you can then run build-all.bat to produce a richer and more capable web exe that's more capable of supporting your App's requirements.

If your development team is comprised of front-end and back-end .NET developers, back-end .NET developers can be responsible for maintaining a customized web .exe containing all the features, APIs and back-end Services that's needed for front-end developers to implement your company's App requirements where they're able to develop several different Web Apps using the same web exe. Essentially similar to front-end devs using ajax in JavaScript Apps to call back-end APIs, they can use Template Websites to develop server-generated Websites and call .NET APIs directly.
