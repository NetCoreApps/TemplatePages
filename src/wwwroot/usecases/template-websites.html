Template Websites shows the versatility and flexibility made possible with Templates where you're able to develop entire content-rich, database-driven websites, not just without any C# code, but without ever needing to build your App! If your Web App requirements fits within the constraints of a Template Website, there has never been an easier and faster way to develop a Web App in .NET.

Dramatically Simplified

Not having to write any code or perform any app builds dramatically reduces the cognitive overhead and conceptual knowledge required for development where the only thing front-end Web developers need to know is Template's syntax and what filters are available to call. All complexity with C#, .NET, namespaces, references, .dlls, strong naming, packages, MVC, Razor, build tools, etc have been completely eliminated. All Web Developers need to do is run a web .NET Core executable and configure a simple web.settings text file to specify where the website files are and what database they want to connect to (if any). For simple websites the webroot can be inferred and no web.settings is needed.

Because of Template's high-fidelity with JavaScript, developing a Website with Template is more like JavaScript App than C# despite binding directly to .NET APIs behind the scenes.

Ultimate Development Experience

In addition to simplicity, the iterative development experience is unparalleled for a .NET App where for the most part you can just leave the web exe running and add the .html template files needed to build your App. No pre-compilation is required so you can just refresh your page to instantly view modified changes.

Extensible Base

Whilst we've developed a Self-contained .NET Core App (SCD) to illustrate the conecept, the lightweight Template Websites also serves as a good foundation to be able to enhance it with any additional .NET Core or ServiceStack features you need which can be enhanced with custom App filters to enable a richer and more capable web exe that's more tailored to support your App's requirements.

If your development team is comprised of front-end and back-end .NET developers, the back-end developers can be responsible for maintaining a customized web .exe containing all the features, APIs and back-end Services that's needed for front-end developers to implement your company's App requirements where they're able to develop several different Web Apps using the same web exe.
